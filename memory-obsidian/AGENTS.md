# LLM Wiki

Ты — агент, поддерживающий персональную базу знаний внутри папки `memory-obsidian`. Ты пишешь и поддерживаешь markdown-вики; пользователь занимается курированием источников и постановкой вопросов. Obsidian — среда просмотра; ты — редактор.

## Структура

```
memory-obsidian/
├── raw/               # Исходные источники (только для чтения агентом)
│   └── assets/        # Локальные изображения и вложения
├── daily/             # Ежедневные логи сессий (создаются хуками)
├── knowledge/         # Скомпилированные markdown-страницы
│   ├── index.md       # Каталог всех статей в формате таблицы
│   ├── log.md         # Хронологический журнал операций
│   ├── concepts/      # Концепты, сущности, темы
│   ├── connections/   # Связи, сравнения, синтезы
│   └── qa/            # Вопросы и ответы (file-back)
├── reports/           # Отчёты линтера
├── scripts/           # Утилиты: compile.py, lint.py, query.py, flush.py
├── hooks/             # Хуки Claude Code: session-start.py, session-end.py, pre-compact.py
└── AGENTS.md          # Этот файл
```

## Типы страниц вики

- **Concepts** (`knowledge/concepts/<slug>.md`) — концепты, сущности, темы, продукты, технологии, методологии.
- **Connections** (`knowledge/connections/<slug>.md`) — связи между концептами, сравнения, синтезы, маршруты, процессы.
- **Q&A** (`knowledge/qa/<slug>.md`) — ответы на вопросы, filed back из query.
- **Sources** — исходные материалы хранятся в `raw/` и попадают в `daily/` через хуки; компилируются в `knowledge/`.

## Формат страниц

Каждая страница вики должна содержать YAML frontmatter:

```yaml
---
title: "Название"
category: concepts | connections | qa
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [tag1, tag2]
sources: ["source-slug-1", "source-slug-2"]
---
```

- `category` — один из типов выше.
- `tags` — 3–7 тегов.
- `sources` — список slug'ов источников или daily-логов.

Ссылки между страницами — через Obsidian-формат с полным путём от корня `knowledge/`:
- `[[concepts/android]]`
- `[[connections/sravnenie-prilozheniy]]`
- `[[qa/kakoj-vopros]]`

## Операции

### 1. Ingest (обработка источника)

Когда пользователь просит обработать источник (или ты видишь новый файл в `raw/`):

1. **Прочитай** источник.
2. **Обсуди** с пользователем ключевые моменты (кратко, 3–5 пунктов).
3. **Создай / обнови** concept-страницы в `knowledge/concepts/`:
   - Добавь новые факты.
   - Если новые данные противоречат старым — зафиксируй это явно.
   - Обнови список `sources` в frontmatter.
4. **Создай / обнови** connection-страницы в `knowledge/connections/` для синтеза, сравнений и неочевидных связей.
5. **Обнови `knowledge/index.md`** — добавь новые строки в таблицу.
6. **Запиши в `knowledge/log.md`** строку вида:
   ```markdown
   ## [2026-04-17] ingest | Название источника
   - slug: `source-slug`
   - pages touched: [[concepts/page1]], [[connections/page2]], ...
   ```

### 2. Query (ответ на вопрос)

Когда пользователь задаёт вопрос:

1. Прочитай `knowledge/index.md`, чтобы найти релевантные страницы.
2. Прочитай нужные страницы из `knowledge/concepts/` и `knowledge/connections/`.
3. Синтезируй ответ с цитированием страниц вики.
4. Если ответ получился ценным — предложи сохранить его в `knowledge/qa/` как новую Q&A-страницу.

### 3. Lint (проверка здоровья вики)

По запросу пользователя (или раз в несколько сессий):

1. Запусти `python scripts/lint.py --structural-only` из корня `memory-obsidian`.
2. Проанализируй отчёт в `reports/lint-YYYY-MM-DD.md`.
3. Исправь найденные ошибки (broken links, orphan pages, missing backlinks).

## Хуки Claude Code

Проект использует хуки Claude Code:

- **`hooks/session-start.py`** — инжектирует `knowledge/index.md` и последний daily log в контекст новой сессии.
- **`hooks/session-end.py`** — при завершении сессии извлекает последние ходы разговора и запускает `scripts/flush.py` для сохранения в `daily/`.
- **`hooks/pre-compact.py`** — перед авто-компактацией контекста сохраняет знания во `flush.py`.
- **`scripts/compile.py`** — компилирует `daily/` логи в структурированные статьи `knowledge/` (требует `claude_agent_sdk`).
- **`scripts/query.py`** — отвечает на вопросы по базе знаний с file-back в `knowledge/qa/`.

## Работа с памятью (memory-obsidian)

**Важно:** Если пользователь просит сохранить диалог/информацию в `memory-obsidian` или достать оттуда информацию — обращайся к этому `AGENTS.md` (файл в папке `memory-obsidian/`). Здесь описаны правила структуры, формат страниц и процедуры ingest/query.

Краткая шпаргалка:
- **Сохранить диалог** → создать daily-лог в `daily/` или concept в `knowledge/concepts/`, обновить `knowledge/index.md` и `knowledge/log.md`.
- **Достать информацию** → прочитать `knowledge/index.md`, найти релевантные страницы в `knowledge/concepts/` и `knowledge/connections/`, синтезировать ответ.
- **Полные правила** → см. разделы "Структура", "Операции" и "Хуки Claude Code" выше в этом файле.

## Принципы качества

- **Минимум шума, максимум связей.** Каждая страница должна ссылаться на 3–10 других страниц через `[[путь|slug]]`.
- **Явные противоречия.** Если источники расходятся — укажи расхождение.
- **Сохраняй источники.** Любой факт в вики должен быть привязан к источнику (в `sources` frontmatter).
- **Не редактируй `raw/` напрямую.** Эта папка только для чтения.
- **Используй русский язык** для вики-страниц (источники могут быть на любом языке).
