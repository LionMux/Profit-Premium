# NEXTAUTH_URL и несовпадение порта dev-сервера

## Проблема

При разработке Next.js с NextAuth.js v5 (Auth.js) dev-сервер может запускаться на разных портах (`3000`, `3001`, `3002`...), если основной порт занят. `NEXTAUTH_URL` в `.env.local` зафиксирован на `http://localhost:3000`.

Когда авторизованный пользователь нажимает ссылку на `/login` (например, кнопку «Личный кабинет» на лендинге):
1. Middleware видит активную сессию (`req.auth`)
2. Редиректит с `/login` на `/dashboard`
3. Но NextAuth использует `NEXTAUTH_URL` для некоторых редиректов
4. Пользователь попадает на `http://localhost:3000/dashboard`, где нет сервера
5. **Результат: 404 (Not Found)**
6. Пользователь чистит куки, чтобы «сбросить» состояние

## Решение

**Вариант 1 (рекомендуется для разработки):** фиксировать порт dev-сервера:
```bash
npm run dev -- -p 3000
```

**Вариант 2:** отключить оптимизацию изображений Next.js, чтобы избежать таймаутов на внешних URL:
```js
// next.config.js
images: {
  unoptimized: true,
}
```

**Вариант 3:** заменить все внешние изображения на локальные, чтобы страницы не падали при проблемах с сетью.

## Правило

> Всегда запускать dev-сервер на фиксированном порте, совпадающем с `NEXTAUTH_URL`. Если порт 3000 занят — освободить его (`taskkill /F /IM node.exe`) или обновить `NEXTAUTH_URL`.
